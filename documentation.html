Dokumentacja

1. Pobranie plików

Pobierz zipa. Pózniej trzeba go będzie rozpoakować i wgrac na serwer.
Znajdują się tam libki które umożliwiają uruchomienie aplikacji na systemie Linux, gdzie wersja Node to 12.
Poniżej przedstawimy możliwą instalację, korzystając z Heroku (serwera, który udostępnia nam dużo możliwości i przy pomocy Atlas Mongo(bazy danych potrzebnej do działania aplikacji)

2. Utworzenie darmowej bazy MongoDb przy pomocy https://account.mongodb.com

  • Tworzymy konto wchodząc na link: https://account.mongodb.com/account/register
  • Po utworzeniu konta, tworzymy darmowy klaster. Strona powinna nas pokierowac jak to zrobić
  • Czekamy, aż stworzy się klaster
  • W zakładce "Clusters" pojawi się nasz nowo stworzony klaster, klikamy w "Collections"
  • Następnie klikamy "Add my own data", wpisujemy nazwę: "meetuper" i nazwę kolekcji np. "test"
  • Wchodzimy w zakładkę "Clusters", tam gdzie wcześniej i szukamy zielonego przycisku Connect Cluster i przyciskamy.
  • Zaznaczamy opcje: "Allow Access From Anywhere"
  • Tworzymy nowego użytkownika poniżej
  • Następnie przyciskamy "Choose a connection method"
  • Wybieramy metodę "Connect your application"
  • Driver ma być ustawiony na Node, następnie kopiujemy podany link podmieniając "< password>" na wpisane wcześniej hasło i "< dbname>" na nazwę bazy danych: meetuper

2. Utworzenie darmowego konta na Heroku

  • Rejestrujemy się wchodząc na link: https://signup.heroku.com/. Wypełniamy dane, wybieramy język inny niż podane, ale też to można później zmienić.
  • Tworzymy nową aplikację: Create new app
  • Wpisujemy nazwę aplikacji i klikamy Stwórz
  • Przechodzimy do zakładki Settings, następnie szukamy przycisku Reveal Config Vars, klikamy
  • Dodajmey kilka wartości: Key: NODE_ENV Value: production
    Key: MONGO_URI Value: wklejamy tutaj skopiowany i uzupełniony adres bazy, którą wcześniej utworzyliśmy
    Key: BUILDPACK_URL Value: https://github.com/ph3nx/heroku-binary-buildpack.git
    Key: PATH Value: /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/app/bin
  • Następnie przechodzimy do Deployu aplikacji, więc wchodzimy do zakładki Deploy i na początku musimy zainstalować Heroku CLI
  • przechodzimy kolejne punkty aż przejdziemy do komendy: heroku git:remote -a nazwa-twojej-aplikacji. Wtedy musimy rozpakować pobranego zipa do powstałego folderu(nazwa folderu to nazwa aplikacji). W tym folderze powinień być folder bin i Procfile
  • Jeżeli już się wszystko zgadza i jest rozpakowane w tym folderze to pądążamy za kolejnymi krokami czyli robimy git add itd. Te kroki mogą trochę zająć czasu
  • Po tych krokach aplikacja będzie działać
  • Po utworzeniu konta, tworzymy darmowy klaster. Strona powinna nas pokierowac jak to zrobić
  • Czekamy, aż stworzy się klaster
Następnie zakładamy konto na heroku, tworzymy sobie nową stronę.
Ustawiamy sobie Reveal Config: takie jak Path, Node_env, Mongo_URI pobrane z wcześniej utworzonej bazy.
Ustawiamy buildPacka
Pobieramy heroku CLI zgodnie z instrukcją:
Kiedy mamy już folder z naszą aplikacja, tam rozpakowujemy zipa, tak aby w głównym folderze był plik Procfile.
Wgrywamy tak jak napisano w instrukcji heroku: git add. git commit i git push heroku master
Ważne żeby przed tym upewnić się że ustawiliśmy BuildPacka i Reveal Config.
